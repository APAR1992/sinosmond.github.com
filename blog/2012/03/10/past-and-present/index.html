过去和现在

Mar 10th, 2012

过去的选择

  • 喜欢 Dotclear 胜过 WordPress
  • 喜欢结构化文档的理念
    • 开源文档世界中的 DocBook 虽好，但 XML 书写繁琐
    • 轻量级的文本标记语言： markdown、 textile、 reST、 txt2tags …
    • 各种 Wiki 标记语言： Creole、 dokuwiki、 Mediawiki …
    • 轻量级的文本标记语言和各种 Wiki 标记语言虽然简单，但语义表达能力弱，写 BLOG 等一般的短文章也足够用了
    • AsciiDoc：折中的选则，架起了文本标记语言和 DocBook 的桥梁，基本实现了 DocBook 的语义表达
  • 逐渐更喜欢不带关系数据库的个人站点应用，因为备份简单（rsync）
    • 最初使用 txt2tags
    • 后转向 Dokuwiki

痛苦的过去

  • 曾经注册过自己的域名

  • 曾经在国内和国外购买过虚拟主机空间

    无奈皆因家中琐事繁多，无暇打理，导致域名被抢注、主机空间因未续费而被停用

如今的选择

  • 使用 github 免费的二级域名，免去因忘记域名续费而被抢注的困扰
  • 使用 github 提供的 300M 免费空间，对于一般情况也足够了，免去因忘记空间续费而被停用的困扰
  • 使用 git 的版本控制替代 wiki 的版本控制，在 github 上 ** BLOG 即 REPO **
  • 使用 markdown 语法替代以前的 DokuWiki 语法书写文档
  • 在 github 上创建个人站点的工具主要是 Octopress 和 Jekyll-Bootstrap，因后者还不太成熟遂选择前者
